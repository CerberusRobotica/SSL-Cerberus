<br>

## Estrutura do Projeto

Seria interessante que cada Projeto tivesse uma branch especifica

<br>

## Padrão de Commits

Vamos seguir um padrão de commits para cada Projeto (IRIS, HADES, HERMES, CARONTE).

### Formato

    <PROJETO>(<tipo>) - <descrição curta e clara>

Exemplo: 

    IRIS(feat) - Criação do backend

### Tipos

    * feat – Nova funcionalidade
    * fix – Correção de bug
    * refactor – Refatoração sem alterar comportamento
    * docs – Documentação
